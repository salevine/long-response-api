Test Response Time API

A simple API to test response times. Specify a wait time in milliseconds and the API will respond after that delay.

Endpoint

GET POST PUT /api/testResponseTime

Parameters

Name Type Required Description
testTime integer Yes Time to wait in milliseconds (0 - 300000)

Response Codes

200 Success - Wait time completed
400 Bad Request - Missing or invalid testTime parameter
405 Method Not Allowed - Only GET, POST, PUT supported

Success Response

{
  "message": "Wait Time Reached: 2000",
  "requestedWaitMs": 2000,
  "startTime": "2024-01-15T10:30:00.000Z",
  "endTime": "2024-01-15T10:30:02.005Z",
  "elapsedMs": 2005
}

Response Fields

Field Description
message Confirmation message with requested wait time
requestedWaitMs The wait time you requested (milliseconds)
startTime ISO 8601 timestamp when the request was received
endTime ISO 8601 timestamp when the response was sent
elapsedMs Actual elapsed time in milliseconds (includes processing overhead)

Examples

GET Request

curl "https://long-response-api.vercel.app/api/testResponseTime?testTime=2000"

POST Request

curl -X POST "https://long-response-api.vercel.app/api/testResponseTime" \
  -H "Content-Type: application/json" \
  -d '{"testTime": 2000}'

PUT Request

curl -X PUT "https://long-response-api.vercel.app/api/testResponseTime" \
  -H "Content-Type: application/json" \
  -d '{"testTime": 2000}'

Notes

  • Maximum wait time is 300,000ms (5 minutes)
  • No authentication required
  • For POST/PUT requests, testTime can be passed in the request body or as a query parameter
